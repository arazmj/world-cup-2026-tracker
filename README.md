# World Cup 2026 Tracker

A fast, offline-friendly web app to follow the **FIFA World Cup 2026**. Type match
scores and watch group standings, third-place qualification and the full knockout
bracket update live.

**Live site:** https://www.amirrazmjou.com/world-cup-2026-tracker/
(also at https://arazmj.github.io/world-cup-2026-tracker/)

## Features

- **Prefilled with the real results so far** — the whole group stage and the
  Round-of-32 games played through June 29, 2026; you predict the rest.
- Real 2026 draw — all 48 teams across 12 groups, with country flags.
- Group standings with the **full FIFA tiebreakers**: points → goal difference →
  goals for → head-to-head (points/GD/GF among level teams) → fair play → drawing of lots.
- Ranks the 12 third-placed teams and qualifies the best eight.
- Knockout bracket (Round of 32 → Final) with the official **Annex C** third-place
  slotting, automatic advancement and penalty-shootout winners for drawn matches.
- Autosaves in your browser; **Export / Import** a JSON file and **share a link**
  that encodes the whole tournament.
- Light / dark theme, responsive, keyboard accessible.

## Accounts & sign-in

Sign in to save your predictions to your account and pick them up on any device. Signed out,
predictions autosave in your browser. The official results feed is shared and read-only.

- **Microsoft** — works out of the box (Microsoft Entra SPA app; the public client id is baked in).
- **Google** — implemented; turn it on by creating a Google OAuth **Web** Client ID, adding your
  site origin(s) to its *Authorized JavaScript origins*, and building with
  `VITE_GOOGLE_CLIENT_ID=<client-id>`. The "Continue with Google" button then activates.
- **Apple** — *Sign in with Apple* requires a paid Apple Developer account; the button is shown
  but disabled until configured.

Signed-in predictions sync to Azure Blob Storage through an authenticated Function endpoint
(`/api/predictions`), which validates the bearer token against Microsoft Graph / Google userinfo
and stores one private blob per user.

### Cloud resources (Azure resource group `wc2026-tracker-rg`)
- Function App `wc2026-results-eaed0e` — hourly results scraper + `/api/predictions` sync API.
- Storage `wc26trackeaed0e` — public `data/results.json`, private `users/` predictions.
- Microsoft Entra app *World Cup 2026 Tracker* (client id `cd46d082-3643-45b4-bb22-3ece538b5695`).

## Develop

```bash
npm install
npm run dev       # start the dev server
npm test          # run the logic + render tests (Vitest)
npm run build     # type-check + production build
npm run preview   # preview the production build
```

## Deploy

Every push to `main` is built and published to GitHub Pages by the GitHub Actions
workflow in `.github/workflows/deploy.yml` (Pages source: GitHub Actions). The Vite
`base` is `/world-cup-2026-tracker/` to match the repository name.

## Data

`src/data/*.json` (teams, schedule, knockout tree and the 495-row Annex C table) and
the flag SVGs in `public/flags/` are generated by `npm run gen-data`
(`scripts/gen-data.py`). `src/data/seed.json` holds the results of games already played
(built by `scripts/build_seed.py`). The tiebreaker and bracket logic in `src/lib/` is
covered by unit tests in `src/test/`.
